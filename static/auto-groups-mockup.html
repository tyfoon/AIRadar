DESIGN NOTES
  • Visueel onderscheid: auto-groups hebben altijd een ✨ indicator, een "system" badge indien aangemaakt door AI-Radar, en pastel gekleurde iconen. Manual groups zijn kleurloos-default.
  • Source indicator op member: auto = door rules toegevoegd · handmatig = user added · uitgesloten = user overrode.
  • Kopiëren als custom: "Copy as custom group" — user kan een system group klonen en de regels aanpassen. Handig voor edge cases zonder het system origineel te breken.
  • Empty state: een auto-group met 0 members blijft zichtbaar (opaque 70%) zodat de user weet dat er "nog geen" devices van dat type zijn. Niet verwijderen.
  • Onboarding: eerste visit na implementatie toont een banner: "AI-Radar heeft 12 auto-groups voor je samengesteld op basis van je 45 devices. Bekijk ze →"
  • Rule editor (later): voor user-created auto-groups komt er een inline editor. System groups zijn read-only zodat we ze kunnen upgraden over releases heen zonder user config te breken.
  • Koppeling met policies: de auto-groups worden scope-selecteerbaar op Rules en Content page. Een policy "Block Camera's van internet" geldt automatisch voor elke toekomstige camera — big value prop voor ZERO-config bij nieuwe devices.
